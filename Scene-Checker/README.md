# MAYAシーンバリチェッカーツール
・Mayaのシーン内にあるオブジェクトを走査し、次の工程に不都合が生じる問題点を自動でチェックするツール
・オブジェクトの初期名、原点ずれ、Ngon、凹面、ポリゴン数超過（仕様書がある場合）をそれぞれボタン一つでまとめて確認することができる。

## 機能
- 初期名（pCube,pSphere等）のまま残っているオブジェクトを検出
- 原点からずれた位置にあるオブジェクトを検出（カメラは除外される）
- N-gon （５以上頂点を持つフェース）を持つオブジェクトを該当面番号付きで検出
- 凹面を持つオブジェクトを該当面番号付きで検出
- シーン全体のポリゴン数を集計し、指定した上限を超過として検出
- 各ボタンをUIのチェックボタンから個別に実行

## 技術的に工夫した点
- チェック項目ごとにメソッドを分割した。後から機能を足しやすい構造にした
- N-gon・凹面の検出には、従来使ってきたcmdsではフェース単位の検出ができないため、OpenMayaAPI（MIｔMeshPolygon）を導入した。
- 検出結果はオブジェクトのどの面かを明示することでどこを修正すればいいかを明確にしユーザー視点の設計を心掛けた。
- ポリゴン数チェックでは上限は入力欄から指定でき、数字以外が入力された際に例外処理ではじく。

## 苦戦したところ
- **cmdsとの違い** : 設計当初はcmdsしか使ったことがなかったために、面ごとの処理が必要になった際（N-gonや凹面）にどの面かの情報が取り出せなかった為新しくAPIを使ったが、リファレンスを見ながらの作業が非常に困難を極めたが何とか気合で乗り切ることができた。cmdsとAPIの設計思想の違いを理解する必要があった。
- **反復子（イテレーター）の仕組みの理解** : APIの反復子はFORループと違い、現在の位置からnext()で進める必要がある点で、理解に時間がかかった。二回いてレートに触れる機会ができたので、実際に手を動かしながら理解することができた。
